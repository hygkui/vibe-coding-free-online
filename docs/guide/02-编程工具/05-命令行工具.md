# 命令行工具

## 为什么命令行对 Vibe Coding 很重要

虽然 Vibe Coding 强调用自然语言与 AI 交互，但命令行仍然是一个不可或缺的工具。命令行可以让你更高效地管理项目、运行脚本、安装依赖，与各种开发工具交互。

很多 AI 生成的项目最终都需要通过命令行来运行和部署。因此，掌握基本的命令行操作是 Vibe Coding 学习者的必修课。

## 基础命令行操作

### 常用命令

以下是开发中最常用的命令行命令：

**cd**（Change Directory）。切换当前目录。例如：cd my-project 进入 my-project 文件夹。cd .. 返回上一级目录。cd / 切换到根目录。

**ls**（List）。列出当前目录的文件和文件夹。在 Windows 上是 dir 命令。

**pwd**（Print Working Directory）。显示当前所在路径。在 Windows 上是 cd 命令。

**mkdir**（Make Directory）。创建新文件夹。例如：mkdir my-project 创建一个名为 my-project 的文件夹。

**rm**（Remove）。删除文件或文件夹。例如：rm file.txt 删除文件。rm -rf folder 删除文件夹及其内容（谨慎使用）。

**cp**（Copy）。复制文件或文件夹。例如：cp source.txt destination.txt。

**mv**（Move）。移动或重命名文件。例如：mv old.txt new.txt 重命名文件。

### 文件操作

查看文件内容：

cat filename 显示文件全部内容。head -n 10 filename 显示文件前 10 行。tail -n 10 filename 显示文件后 10 行。

编辑文件：

Windows 上可以用 notepad 或 code（VS Code）。Mac 和 Linux 上可以用 nano、vim 或 code。

### 包管理器

包管理器是管理和安装软件包的工具，不同系统有不同的选择：

**npm**（Node Package Manager）。随 Node.js 一起安装，是最常用的 JavaScript 包管理器。使用 npm install 安装依赖，npm run 执行脚本。

**pnpm**。npm 的替代品，速度更快，磁盘占用更少。使用 pnpm install 安装依赖。

**Homebrew**。Mac 上的包管理器，用于安装命令行工具和应用程序。使用 brew install xxx 安装软件。

**Chocolatey**。Windows 上的包管理器。使用 choco install xxx 安装软件。

## 版本控制：Git

Git 是目前最流行的版本控制系统，几乎所有的代码项目都使用 Git 管理。

### Git 基础

初始化仓库：git init 在当前目录初始化一个新的 Git 仓库。

克隆仓库：git clone url 从远程仓库克隆代码。

添加文件：git add filename 将文件添加到暂存区。git add . 添加所有修改。

提交：git commit -m "message" 提交暂存区的文件。

查看状态：git status 查看当前仓库状态。

查看历史：git log 查看提交历史。

### 远程仓库

添加远程仓库：git remote add origin url 将远程仓库关联到本地仓库。

推送：git push origin main 将本地提交推送到远程仓库。

拉取：git pull origin main 从远程仓库拉取最新代码。

### 分支管理

创建分支：git branch branch-name 创建新分支。

切换分支：git checkout branch-name 切换到指定分支。

创建并切换：git checkout -b branch-name 创建并切换到新分支。

合并分支：git merge branch-name 将指定分支合并到当前分支。

### Git 图形化工具

虽然命令行可以完成所有 Git 操作，但图形化工具可能更容易上手：

GitHub Desktop 是 GitHub 官方提供的图形化 Git 工具，界面友好，适合初学者。Sourcetree 是一个功能强大的免费 Git 客户端。VS Code 内置 Git 支持，可以直接在编辑器中完成大部分 Git 操作。

## 开发环境管理

### Node.js 版本管理

Node.js 有不同的版本，项目可能需要特定的 Node.js 版本。使用版本管理器可以轻松切换版本：

**nvm**（Node Version Manager）是最流行的 Node.js 版本管理工具。使用 nvm install 安装指定版本，nvm use 切换版本。

**n** 是另一个选择，更简单但功能稍少。

### 环境变量

环境变量用于存储配置信息，如 API 密钥、数据库地址等。

查看环境变量：在 Linux/Mac 上用 echo $VAR_NAME。在 Windows 上用 echo %VAR_NAME%。

设置环境变量：export VAR_NAME=value（Linux/Mac）。set VAR_NAME=value（Windows）。

在项目中，通常使用 .env 文件来管理环境变量，项目代码通过特定的库读取这些值。

## 脚本和自动化

### npm 脚本

在 package.json 中可以定义各种脚本：

```json
{
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "build": "webpack",
    "test": "jest"
  }
}
```

运行脚本：npm run script-name。

### 常用脚本场景

自动格式化代码：添加 prettier --write . 脚本。

代码检查：添加 eslint --ext .js,.vue src 脚本。

运行测试：添加 jest 或 vitest 脚本。

构建项目：添加 webpack 或 vite build 脚本。

## 实用技巧

提高命令行效率的几个技巧：

善用 Tab 补全。输入文件名或命令时，按 Tab 键可以自动补全，减少输入量和错误。

使用上下箭头键可以快速选择之前执行过的命令。

使用 Ctrl+R 可以搜索命令历史。

为常用命令创建别名。在 .bashrc 或 .zshrc 中添加 alias short="long command"。

## 学习建议

对于 Vibe Coding 学习者，我们建议：

先掌握基础命令：cd、ls、mkdir、rm、cp、mv、cat、echo。

然后学习 Git 基本操作：init、add、commit、push、pull、branch、merge。

了解包管理器的基本使用：npm install、npm run。

不要死记硬背命令。遇到不确定的命令可以查文档，用多了自然就记住了。

遇到问题时善用搜索引擎。命令行问题几乎都能找到解决方案。
