# 工具实战

## 实战概述

前面的章节介绍了各种 Vibe Coding 工具，但理论需要与实践结合。在这一章中，我们将通过具体的实战项目来巩固所学知识。

每个实战项目都会综合运用多种工具，让你体验完整的 Vibe Coding 开发流程。

## 实战一：个人待办事项应用

### 项目目标

创建一个个人待办事项管理应用，具备以下功能：添加新的待办事项；标记事项为已完成或未完成；删除不需要的待办事项；将数据保存到本地存储；响应式设计，支持移动端和桌面端。

### 技术栈选择

本项目使用纯前端技术：HTML、CSS、JavaScript。不需要后端服务器，也不需要数据库。所有数据存储在浏览器的 LocalStorage 中。

### 开发步骤

第一步，创建项目基础结构。在命令行中创建项目文件夹和文件。

第二步，编写 HTML 结构。使用自然语言向 AI 描述你需要的页面结构，让 AI 生成基础 HTML 代码。

第三步，添加样式。使用 CSS 实现设计效果。告诉 AI 你想要的视觉风格，让 AI 生成对应的 CSS 代码。

第四步，实现交互逻辑。这是核心步骤，需要和 AI 详细讨论每个功能的实现方式。

第五步，数据持久化。实现 LocalStorage 的读写逻辑，确保数据不会丢失。

第六步，测试和优化。测试所有功能，修复发现的问题，优化代码结构。

### 关键对话示例

以下是一些关键的对话示例：

```
请帮我创建一个待办事项应用的 HTML 文件。需要包含：一个标题"我的待办事项"；一个输入框和一个添加按钮；一个无序列表用于显示待办事项；每个待办事项需要包含复选框、文本内容、删除按钮。
```

```
现在请帮我写 CSS 样式。要求：整体采用简洁的白色和浅灰色配色；卡片式的待办事项设计；复选框和删除按钮要有明显的视觉反馈；支持移动端响应式布局。
```

```
接下来实现 JavaScript 功能：
1. 点击添加按钮或按回车键时，将输入框的内容添加到列表
2. 每个待办事项要有独立的 ID
3. 点击复选框切换完成状态，已完成的文字要有删除线效果
4. 点击删除按钮移除该项
5. 所有数据保存到 LocalStorage，页面刷新后数据不丢失
```

## 实战二：天气查询应用

### 项目目标

创建一个天气查询应用，用户输入城市名称后显示该城市的天气信息。

### 技术栈选择

前端使用 HTML、CSS、JavaScript。后端使用免费的天气 API 获取数据。

### 使用的工具

本项目将综合使用以下工具：VS Code 作为代码编辑器；GitHub Copilot 或类似 AI 插件辅助编码；Open-Meteo 或其他免费天气 API；Git 管理代码版本。

### 开发步骤

第一步，申请 API 密钥。注册天气 API 服务，获取 API 密钥。

第二步，设计界面。设计一个简洁的天气查询界面，包含搜索框和结果显示区域。

第三步，实现 API 调用。使用 fetch 或 axios 调用天气 API。

第四步，处理和展示数据。将 API 返回的 JSON 数据解析并展示在页面上。

第五步，错误处理。处理网络错误、API 错误、无效输入等情况。

第六步，优化体验。添加加载状态、缓存机制、快捷城市选择等功能。

### 关键对话示例

```
我想要创建一个天气查询应用。使用 Open-Meteo 免费 API（不需要 API 密钥）。页面包含：一个搜索框，一个搜索按钮，一个显示天气信息的区域。显示的信息包括：城市名称、当前温度、天气状况、最高最低温度、风速。
```

```
Open-Meteo API 的使用方式是：https://api.open-meteo.com/v1/forecast?latitude={纬度}&longitude={经度}&current=temperature_2m,weather_code,wind_speed_10m

请帮我实现：
1. 用户输入城市名称后，需要通过地理编码 API 获取城市的经纬度
2. 然后调用天气 API 获取天气数据
3. 将结果显示在页面上
```

## 实战三：个人博客系统

### 项目目标

创建一个功能完整的个人博客系统，包括：文章列表展示；文章详情页；文章分类和标签；简单的搜索功能；响应式设计。

### 技术栈选择

本项目使用静态站点生成方案：VuePress 或 VitePress 作为框架；Markdown 编写文章；GitHub Pages 免费部署。

### 使用的工具

本项目将综合使用以下工具：VS Code + VuePress 插件；GitHub 托管代码和部署；ChatGPT/Claude 辅助开发；Git 管理版本。

### 开发步骤

第一步，初始化项目。使用 VitePress 模板初始化项目。

第二步，配置站点。修改配置文件，设置站点标题、描述、导航栏、侧边栏等。

第三步，编写内容。创建博客文章，使用 Markdown 格式。

第四步，定制主题。修改样式，实现个性化的博客外观。

第五步，添加功能。实现文章分类、标签、搜索等功能。

第六步，部署上线。使用 GitHub Pages 部署博客。

### 关键对话示例

```
我想用 VitePress 创建一个个人博客。请帮我：
1. 初始化一个基本的 VitePress 项目结构
2. 配置站点标题为"我的技术博客"，描述为"记录学习笔记和技术分享"
3. 配置导航栏包含首页、分类、关于
4. 配置侧边栏显示文章列表
```

## 实战四：Chrome 扩展开发

### 项目目标

创建一个 Chrome 浏览器扩展，实现以下功能：在浏览器工具栏显示图标；点击图标弹出一个小窗口；小窗口可以执行一些快捷操作，如快速笔记、网址收藏等。

### 技术栈选择

Chrome 扩展使用标准的 Web 技术：HTML、CSS、JavaScript；Chrome APIs；Manifest V3 配置。

### 开发步骤

第一步，创建扩展基础结构。创建清单文件（manifest.json）和相关文件。

第二步，设计 popup 界面。设计扩展点击后弹出的界面。

第三步，实现核心功能。编写 JavaScript 实现具体功能。

第四步，测试扩展。加载扩展到 Chrome 中进行测试。

第五步，优化和完善。改进 UI，修复 bug，添加新功能。

### 关键对话示例

```
我想创建一个 Chrome 扩展，功能是在新标签页显示一个欢迎页面，显示当前时间、一句励志名言、快捷便签功能。请帮我创建：
1. manifest.json 配置文件
2. 新标签页的 HTML、CSS、JS 文件
3. 实现便签的添加、删除和本地存储功能
```

## 实战五：微信小程序开发

### 项目目标

创建一个简单的小程序项目，如：待办事项、笔记应用、天气查询等。

### 技术栈选择

微信小程序使用微信官方开发框架：WXML 模板语言；WXSS 样式语言；JavaScript 逻辑层；微信云开发或自定义后端。

### 开发步骤

第一步，注册小程序。在微信公众平台注册小程序账号。

第二步，安装开发工具。下载并安装微信开发者工具。

第三步，创建项目。使用开发者工具创建新项目。

第四步，编写页面。按照设计实现各个页面。

第五步，实现功能。编写业务逻辑代码。

第六步，测试和提交。使用开发者工具测试，然后提交审核。

### 关键对话示例

```
我想创建一个微信小程序，功能是记录每日开销。要求：
1. 可以添加开销记录，包括金额、分类、备注
2. 按日期显示开销列表
3. 显示当日和本月的总开销
4. 可以按分类筛选查看
请帮我创建项目结构和基础代码。
```

## 实战总结

通过这些实战项目，你应该已经掌握了 Vibe Coding 的基本流程和工具使用。

每个项目的关键成功因素包括：清晰的需求描述；合理的任务分解；充分的测试验证；持续的代码优化。

建议在完成这些项目后，尝试自己设计并实现一些新的项目。实践是最好的老师，只有不断实践，才能真正掌握 Vibe Coding 的技能。
