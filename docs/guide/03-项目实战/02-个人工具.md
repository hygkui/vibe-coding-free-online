# 个人工具

## 个人工具开发的价值

个人工具是指为了解决个人需求而开发的软件应用。这类项目通常规模不大，但能显著提高个人效率。学习开发个人工具是 Vibe Coding 的最佳起点，因为需求明确、反馈直接、改进动力强。

## 自动化脚本开发

### 用途分类

自动化脚本可以处理各种重复性工作：文件操作，如批量重命名、格式转换、数据整理。数据处理，如 Excel 表格自动化、报告生成。系统操作，如定时任务、文件备份、清理垃圾。网络任务，如自动签到、数据采集、消息推送。

### 技术选择

对于简单的脚本任务，Python 是最佳选择。它的语法简洁，生态丰富，几乎什么都有现成的库。

对于浏览器自动化，可以使用 Python 的 Playwright 库或 JavaScript 的 Puppeteer/Playwright。

对于系统级任务，Shell 脚本（Windows 用 PowerShell）更方便。

### 开发示例

以下是一些常见的自动化脚本示例：

批量文件处理脚本可以帮你自动处理大量文件。比如将某个文件夹内的所有图片转换为 WebP 格式，或者批量修改文件名添加日期前缀。

数据报告脚本可以自动从多个数据源获取数据，生成汇总报告。比如每日报表、周报自动生成。

监控系统脚本可以监控系统状态，在异常时发送通知。比如服务器磁盘空间监控、网站可用性检测。

### 对话示例

与 AI 对话生成自动化脚本时，可以这样描述：

```
我想要一个 Python 脚本，功能是监控网站的可用性。要求：定期访问指定的 URL，记录响应时间和状态码；当响应时间超过阈值或状态码不是 200 时，发送邮件通知；将监控结果保存到日志文件。
```

## 效率工具开发

### 剪贴板增强

默认的剪贴板功能有限。开发一个剪贴板增强工具可以实现：剪贴板历史，记录最近复制的内容；剪贴板管理，对历史内容进行搜索和整理；剪贴板预设，保存常用的文本片段。

### 快速启动工具

开发一个快速启动工具可以提高工作效率：文件快速搜索，快速找到需要的文件；应用快速启动，一键启动常用应用；快捷命令，定义常用操作的快捷方式。

### 笔记整理工具

个人笔记管理是常见需求。可以开发以下功能：笔记采集，快速记录想法和片段；标签管理，通过标签组织和查找笔记；知识图谱，展示笔记之间的关联。

## 浏览器扩展开发

### 常用功能

浏览器扩展可以极大地扩展浏览器的能力：广告拦截，屏蔽网页广告。页面增强，添加额外功能到任意网页。数据采集，从网页提取特定数据。自动化，自动执行网页操作。

### 开发基础

Chrome 扩展的核心是 Manifest V3 配置文件和相关的 HTML/CSS/JS 文件。

一个简单的扩展包含：manifest.json 定义扩展的基本信息和权限。popup.html 弹出窗口的界面。popup.js 弹出窗口的逻辑。content scripts 在页面中执行的脚本。

### 开发示例

创建待办事项扩展：

```
创建一个 Chrome 扩展，功能是在工具栏显示一个图标，点击后弹出待办事项面板。要求：可以添加、编辑、删除待办事项；支持标记完成；数据存储在 chrome.storage.sync 中；支持同步，在不同设备间保持数据一致。
```

## 命令行工具开发

### 用途分类

命令行工具可以完成各种自动化任务：文件管理，批量操作文件和文件夹。代码开发，代码生成、格式化、重构。系统管理，系统信息查看、进程管理、服务控制。

### 开发示例

创建一个文件管理工具：

```
开发一个命令行工具，功能是快速管理项目文件。包含以下命令：list 列出当前目录的文件和文件夹；search 在文件中搜索指定内容；replace 批量替换文件中的文本；clean 清理临时文件和缓存。
```

### 参数设计

好的命令行工具应该有清晰的参数设计：使用标准参数格式（如 -v 表示版本，-h 表示帮助）；提供详细的使用说明；支持配置文件定制默认行为。

## 数据可视化工具

### 用途分类

数据可视化工具可以将数据转化为直观的图表：个人仪表盘，展示关键指标的变化趋势。数据报告，将数据转化为可读的报告。实时监控，实时展示系统或业务的运行状态。

### 技术选择

前端可视化可以使用 Chart.js、ECharts、D3.js 等库。后端需要提供数据接口，通常使用 Node.js/Python + 数据库。

### 开发示例

个人财务仪表盘：

```
创建一个个人财务仪表盘 web 应用。要求：可以记录收入和支出；按月、周、日展示收支趋势；按分类显示支出分布；支持数据导出为 CSV/Excel。
```

## 移动端工具

### 开发选择

个人使用的移动端工具可以选择不同方案：微信小程序，开发简单，但功能受限。PWA 应用，渐进式 web 应用，可以在手机浏览器中使用。React Native/Flutter，需要更多开发工作，但功能完整。

### 常用功能

适合个人使用的移动端工具：习惯追踪，帮助养成好习惯。目标管理，跟踪个人目标的完成进度。快速记账，记录日常开销。灵感收集，随时记录灵感和想法。

## 个人工具开发最佳实践

### 从痛点出发

开发个人工具时，从自己真正痛点出发。不要为了开发而开发，要解决实际问题。

### 保持简单

个人工具不需要一开始就功能完备。先实现核心功能，使用一段时间后再决定是否需要添加更多功能。

### 持续改进

工具是为自己用的，遇到问题随时改进。保持工具的更新，使其越来越符合自己的需求。

### 分享价值

如果你的工具对他人也有帮助，考虑开源分享。一方面可以收到反馈，另一方面也可能帮助到有相同需求的人。
